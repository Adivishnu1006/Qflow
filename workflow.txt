# QFlow - Sprint 1 Summary (Frontend Foundation)

## Project Goal

QFlow is a digital queue management platform that allows customers to join queues remotely and organizations to manage their queues digitally.

There are **two user types**:

* 👤 Customer
* 🏢 Organization

Both use the same application but have different dashboards after authentication.

# Authentication Logic (Future)

Backend decides where to redirect.

Example

```
User

↓

Login

↓

Backend verifies credentials

↓

Returns

{
    token,
    role
}
```

Frontend

```
if(role==="customer")

↓

Customer Home

else

↓

Organization Dashboard
```

No separate login pages.

No role selection during login.

Backend is the source of truth.

# Notes for Backend Developer

1. **Do not change route names** (`ROUTES` constants are considered stable).
2. Registration pages are complete from a UI perspective; wire them to the register APIs.
3. Login UI is complete; replace the temporary navigation with the real login API call.
4. Backend should return the authenticated user's role (customer or organization), and the frontend will redirect to the correct dashboard.
5. Layouts (`PublicLayout`, `AuthLayout`, `CustomerLayout`, `OrganizationLayout`) are shared wrappers. Pages render inside them through React Router's `<Outlet />`.
6. The next shared milestone is implementing:

   * `POST /auth/customer-register`
   * `POST /auth/organization-register`
   * `POST /auth/login`
   * JWT storage
   * Protected routes
